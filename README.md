# Задание 1. LINQ-ЗАПРОСЫ

## Задача 1

### Текст задачи

Разработать консольное приложение с дружественным интерфейсом с возможностью выбора
заданий для работы с «базой данных (БД)», хранящейся в бинарном файле. Перечень полей
(минимум 5), достаточно полно характеризующих заданную в варианте предметную область,
предложить самостоятельно (постараться отразить в перечне полей такие, которые требуют разных
типов данных). Приложение должно выполнять следующие функции:
1. Чтение базы данных из бинарного файла.
2. Просмотр базы данных.
3. Удаление элементов (по ключу).
4. Добавление элементов.
5. Реализация 4 запросов (формулировки запросов придумать самостоятельно). 2 запроса
должны возвращать перечень, 2 запроса одно значение.
В классе должны присутствовать свойства, конструкторы, перегруженный метод ToString(). Весь
функционал приложения реализовать в виде методов вспомогательного класса с помощью LINQзапросов.
Расписание автостанции.

### Алгоритм решения

1. Класс BusSchedule 
Свойства: Характеристики рейса - номер, город, время, цена, места
Конструкторы: По умолчанию и с параметрами
Метод ToString(): Превращает объект в удобную строку для вывода в консоль

2. Класс DatabaseHelper 
LoadDatabase(): Проверяет наличие файла. Если файла нет — создает тестовые города. 
Если есть — читает через BinaryReader и собирает их в список объектов.

SaveDatabase(): Берет список объектов и записывает их поля по порядку в бинарный файл через BinaryWriter.

QueryByDestination() (LINQ): Фильтрует рейсы по названию города и сортирует их по времени отправления.

QueryByAvailableSeats() (LINQ): Находит рейсы с нужным количеством мест и группирует их по типу автобуса.

QueryAverageTicketPrice() (LINQ): Вычисляет среднюю стоимость билета по всей базе одной командой .Average().

QueryTotalAvailableSeatsByDate() (LINQ): Фильтрует рейсы по конкретной дате и суммирует все свободные места через .Sum().

3. Класс Program 

DrawMainMenu(): Рисует графический интерфейс меню и показывает актуальное количество записей в базе.

AddNewRecord(): Собирает данные от пользователя, проверяет их на логику (например, чтобы свободных мест не было больше, чем всего) и создает новый объект.

DeleteRecord(): Запрашивает ключи (номер и время) и удаляет соответствующий рейс из списка.

Методы (Query...): Вызывают аналитические методы из DatabaseHelper и выводят результат поиска на экран.
